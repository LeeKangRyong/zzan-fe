# Clean Code Guide (Woowacourse Style)

이 문서는 우아한테크코스 프리코스의 클린 코드 원칙을 `zzan-fe` 프로젝트에 적용하기 위한 지침입니다. Claude는 코드를 생성할 때 다음 규칙을 엄격히 준수해야 합니다.

## 1. 함수(메서드) 분리
- **단일 책임 원칙**: 함수는 한 가지 일만 해야 합니다.
- **길이 제한**: 한 함수(또는 컴포넌트)의 길이가 15라인을 넘어가지 않도록 노력합니다. 넘는다면 함수 분리를 검토하세요.
- **들여쓰기(indent) 깊이**: `if`, `for`, `while` 등의 들여쓰기 깊이는 최대 **2**까지만 허용합니다. (3이 되면 함수로 분리하세요)

## 2. 네이밍 컨벤션
- **의미 있는 이름**: 변수명에 `data`, `info`, `list` 같은 모호한 단어보다는 실제 역할을 드러내는 이름을 사용합니다.
- **축약 금지**: 의도를 전달할 수 있다면 이름이 길어져도 괜찮습니다. 축약어 사용을 지양하세요.
- **함수명**: 동사로 시작하며 함수의 기능을 명확히 설명해야 합니다.

## 3. 리액트 및 상태 관리 (MVVM 연계)
- **ViewModel 로직 분리**: UI 컴포넌트(View) 내에 비즈니스 로직을 직접 작성하지 않습니다. 모든 로직은 `domains/{domain}/viewmodel/`의 커스텀 훅으로 이동시킵니다.
- **상태 업데이트**: 상태 변경 로직이 복잡해지면 ViewModel 내부에서 작은 함수들로 쪼개어 가독성을 높입니다.

## 4. 리팩터링 및 가독성
- **Else 예약어 금지**: `if` 조건문에서 `else`를 사용하지 않고 조기 반환(Early Return)을 활용합니다.
- **매직 넘버/스트링 제거**: 의미를 알기 어려운 숫자나 문자열은 `shared/constants/` 또는 `Model` 레이어의 상수로 치환합니다.

## 5. TypeScript 및 방어적 프로그래밍
- **타입 지정**: 모든 Props와 함수의 반환값에 타입을 명시하여 `any` 발생을 원천 차단합니다.
- **Null 안전성**: Optional Chaining(`?.`)과 Nullish Coalescing(`??`)을 활용하여 런타임 에러를 방지합니다.
## 6. 테스트 가능성 (Testability)
- **UI와 로직의 분리**: UI 컴포넌트(View)는 비즈니스 로직에 의존하지 않아야 합니다. 로직은 ViewModel(Custom Hook)로 분리하여 로직만 단독으로 테스트할 수 있어야 합니다.
- **순수 함수 활용**: 데이터 변환 로직은 가급적 외부 상태를 참조하지 않는 '순수 함수'로 만들어 `shared/utils/`에 배치하고, 단위 테스트를 수행하기 쉽게 만듭니다.

## 7. 클래스(컴포넌트) 분리 기준
- **필드(상태) 개수**: 하나의 컴포넌트나 ViewModel이 관리하는 상태(useState)가 너무 많아지면(예: 5개 이상), 해당 역할을 작은 단위의 컴포넌트나 커스텀 훅으로 분리하는 것을 고려합니다.
- **주석 없이 코드만으로 설명**: 변수명과 함수명만으로도 이 코드가 무엇을 하는지 알 수 있어야 합니다. 주석은 '무엇'이 아닌 '왜'를 설명할 때만 사용합니다.

## 8. 리액트 전용 클린 코드 (Clean React)
- **Props Destructuring**: Props를 전달받을 때 구조 분해 할당을 사용하여 어떤 값이 쓰이는지 명확히 드러냅니다.
- **명명된 내보내기 (Named Export)**: `export default`보다는 `export const Component = ...` 형태의 Named Export를 사용하여 임포트 시 이름 혼선을 방지합니다.
- **Primitive Props 지향**: 객체 전체를 Props로 넘기기보다, 필요한 원시값(string, number)만 넘겨 컴포넌트 간의 결합도를 낮춥니다.

## 9. 상수화 (Constants)
- **중복 문자열 제거**: 앱 내에서 반복되는 메시지, API 경로, 고정된 수치 등은 모두 상수로 관리합니다.
- **StyleSheet 내 매직 넘버 금지**: `padding: 16` 같은 수치는 `shared/constants/Layout.ts` 등에 정의된 시스템 수치를 따릅니다.
