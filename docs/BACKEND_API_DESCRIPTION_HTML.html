공통 사항

응답 형식

모든 API 응답은 다음 형식을 따릅니다:

{
    "success": true,
    "timestamp": 1705471200000,
    "message": null,
    "data": {
        ...
    }
}
필드타입설명
successboolean요청 성공 여부
timestampnumber응답 시간 (Unix timestamp)
messagestring?에러 메시지 (실패 시)
dataany?응답 데이터

인증 헤더

인증이 필요한 API는 다음 헤더가 필요합니다:

Authorization: Bearer {accessToken}

커서 기반 페이지네이션

요청 파라미터

파라미터타입필수기본값설명
sizeintX10페이지 크기 (1~20)
cursorstringXnull다음 페이지 커서

사용 방법

  1. 첫 번째 페이지: cursor 생략 또는 null
    GET /liquors/{liquorId}/reviews?size=10
  1. 다음 페이지: 이전 응답의 nextCursor 값 사용
    GET /liquors/{liquorId}/reviews?size=10&cursor=01HQXYZ123456789ABCDEF
  1. 마지막 페이지: hasNext: false이면 더 이상 데이터 없음

응답 형식

{
    "items": [
        ...
    ],
    "nextCursor": "01HQXYZ123456789ABCDEF",
    "hasNext": true
}
필드타입설명
itemsarray데이터 목록
nextCursorstring?다음 페이지 커서 (없으면 null)
hasNextboolean다음 페이지 존재 여부

공통 에러 응답

{
    "success": false,
    "timestamp": 1705471200000,
    "message": "에러 메시지",
    "data": null
}
HTTP 상태에러 유형설명
400Bad Request잘못된 요청 파라미터
401Unauthorized인증 필요 또는 토큰 만료
403Forbidden권한 없음
404Not Found리소스를 찾을 수 없음
500Internal Server Error서버 내부 오류

1. 인증 (Auth)

1.1 소셜 로그인 URL 조회

소셜 로그인 페이지 URL을 가져옵니다.

GET /users/auth/{provider}/login-url

Path Parameters

파라미터타입필수설명
providerstringO소셜 제공자 (kakao, google, apple)

Response (200 OK)

{
    "success": true,
    "data": {
        "url": "https://kauth.kakao.com/oauth/authorize?client_id=..."
    }
}

실패 케이스

상태조건메시지
400지원하지 않는 provider“지원하지 않는 소셜 제공자입니다: {provider}”

1.2 소셜 로그인 콜백 처리

소셜 로그인 후 리다이렉트되는 콜백을 처리합니다.

GET /users/auth/{provider}/callback?code={code}

Path Parameters

파라미터타입필수설명
providerstringO소셜 제공자

Query Parameters

파라미터타입필수설명
codestringO소셜 인가 코드

Response (200 OK)

{
    "success": true,
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
}

실패 케이스

상태조건메시지
400지원하지 않는 provider“지원하지 않는 소셜 제공자입니다: {provider}”
401유효하지 않은 code“인가 코드가 유효하지 않습니다”

1.3 소셜 토큰으로 로그인

클라이언트에서 직접 받은 소셜 액세스 토큰으로 로그인합니다.

POST /users/auth/{provider}/login

Path Parameters

파라미터타입필수설명
providerstringO소셜 제공자

Request Body

{
    "accessToken": "소셜_액세스_토큰"
}
필드타입필수설명
accessTokenstringO소셜 제공자에서 발급받은 액세스 토큰

Response (200 OK)

{
    "success": true,
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
}

실패 케이스

상태조건메시지
400accessToken 누락“액세스 토큰은 비어 있을 수 없습니다.”
400지원하지 않는 provider“지원하지 않는 소셜 제공자입니다: {provider}”
401유효하지 않은 소셜 토큰“소셜 액세스 토큰이 유효하지 않습니다”

1.4 토큰 갱신

리프레시 토큰으로 새 액세스 토큰을 발급받습니다.

POST /users/auth/token/refresh

Request Body

(plain text) refreshToken값

Response (200 OK)

{
    "success": true,
    "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
}

실패 케이스

상태조건메시지
401만료된 리프레시 토큰“리프레시 토큰이 유효하지 않습니다.”
401잘못된 토큰 형식“리프레시 토큰이 유효하지 않습니다.”

1.5 로그아웃 (리프레시 토큰 삭제)

DELETE /users/auth/token/refresh

Headers

Authorization: Bearer {accessToken}

Response (200 OK)

{
    "success": true,
    "data": null
}

3.2 주류 검색

GET /liquors/search?keyword={keyword}&page={page}&size={size}

Query Parameters

파라미터타입필수기본값설명
keywordstringO-검색 키워드
pageintX1페이지 번호 (1부터 시작)
sizeintX10페이지 크기

Response (200 OK)

{
    "success": true,
    "data": {
        "items": [
            {
                "id": "01HQ1234567890ABCDEF",
                "liquorName": "느린마을 막걸리",
                "liquorScore": 4.5,
                "liquorImageUrl": "https://cdn.zzan.com/liquor-images/...",
                "liquorType": "탁주"
            }
        ],
        "nextCursor": null,
        "hasNext": false
    }
}
  • 첫 페이지: page 생략 또는 1
  • 응답의 nextCursor: 다음 페이지 번호
    • "nextCursor": "2"
  • 다음 페이지: nextCursor 값을 page에 넣어서 요청
    • GET /....&page=2&size=10
  • 마지막 페이지: hasNext: false, nextCursor: null

10. 스토리지 (Storage)

10.1 Presigned URL 발급

🔐 인증 필요

S3 직접 업로드를 위한 Presigned URL을 발급받습니다.

POST /storage/{prefix}/presigned-url

Path Parameters

파라미터타입필수설명
prefixstringO이미지 종류 (liquor-images, feed-images, user-profile-images)

Request Body

{
    "fileName": "photo.jpg"
}
필드타입필수설명
fileNamestringO업로드할 파일명 (확장자 포함)

Response (200 OK)

{
    "success": true,
    "data": {
        "url": "https://s3.ap-northeast-2.amazonaws.com/bucket/feed-images/abc123.jpg?X-Amz-...",
        "key": "feed-images/abc123.jpg"
    }
}
필드타입설명
urlstring이미지 업로드용 Presigned URL (PUT 요청으로 업로드)
keystring업로드 후 사용할 이미지 경로 (피드 생성 시 사용)

실패 케이스

이미지 업로드 방법

# Presigned URL로 직접 PUT 요청
curl -X PUT \
  -H "Content-Type: image/jpeg" \
  --data-binary @photo.jpg \
  "{presigned_url}"

API 호출 시나리오

시나리오 1: 웹 소셜 로그인(네이티브 X)

1. GET /users/auth/kakao/login-url
   → 카카오 로그인 URL 획득

2. (사용자가 카카오 로그인 완료)

3. GET /users/auth/kakao/callback?code={code}
   → accessToken, refreshToken 획득

4. GET /users/me (Authorization: Bearer {accessToken})
   → 내 정보 확인

시나리오 2: 주류 검색 및 리뷰 작성

1. GET /liquors/search?keyword=막걸리
   → 주류 목록 획득

2. GET /liquors/{liquorId}
   → 선택한 주류 상세 정보 확인

3. GET /liquors/{liquorId}/reviews?size=10
   → 다른 사람들의 리뷰 확인

4. GET /liquors/{liquorId}/reviews/me (인증)
   → 내가 이미 리뷰를 작성했는지 확인

5-a. (리뷰 없음) POST /liquors/{liquorId}/reviews (인증)
   → 새 리뷰 작성

5-b. (리뷰 있음) PUT /liquors/{liquorId}/reviews (인증)
   → 기존 리뷰 수정

시나리오 3: 피드 작성 (이미지 포함)

1. POST /storage/feed-images/presigned-url (인증)
   → 이미지 업로드 URL 및 key 획득

2. PUT {presigned_url} (S3 직접 업로드)
   → 이미지 업로드

3. GET /infra/places/search?keyword=전통주
   → 장소 검색

4. GET /liquors/search?keyword=느린마을
   → 태그할 주류 검색

5. POST /feeds (인증)
   → 피드 생성 (images[].imageUrl에 step 1의 key 사용)

시나리오 4: 지도에서 장소 탐색

1. GET /places?minLongitude=126.9&maxLongitude=127.0&minLatitude=37.5&maxLatitude=37.6
   → 현재 지도 영역의 장소 목록 (마커 표시용)

2. GET /places/{placeId}
   → 선택한 장소 상세 정보

3. GET /feeds/places/{kakaoPlaceId}?size=10
   → 해당 장소의 피드 목록 (첫 페이지)

4. GET /feeds/places/{kakaoPlaceId}?size=10&cursor={nextCursor}
   → 다음 페이지 로드

   → 새 accessToken 획득
